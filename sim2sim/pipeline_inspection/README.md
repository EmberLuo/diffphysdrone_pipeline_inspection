# pipeline_inspection

PX4/Gazebo pipeline inspection navigation stack:

```text
PX4/Gazebo + Mid360
  -> Point-LIO mapping and saved PCD map
  -> NDT/ICP localization on the saved map
  -> Fast-Drone A* global path
  -> path lookahead target
  -> E2E depth policy local control
  -> MAVROS attitude setpoint
```

## Directory Layout

| Directory | Purpose |
|---|---|
| `ros/` | Project ROS packages and launch entrypoints. |
| `third_party/point_lio/` | Point-LIO mapping source used by the thesis SLAM path. |
| `third_party/fast_lio/` | Legacy FAST-LIO mapping fallback. |
| `third_party/fast_drone_250/` | Minimal Fast-Drone planner stack used by A* and EGO baseline. |
| `third_party/fuel/` | FUEL exploration/planning baseline, not linked by default. |
| `third_party/gazebo_plugins/` | Repo-local Gazebo plugins such as Mid360 simulation. |
| `baselines/` | SE3 controller and PX4/MAVROS sanity-check examples. |
| `sim/worlds/` | Gazebo worlds. |
| `sim/models/` | Repo-local model overlays generated by tools. |
| `assets/maps/` | Saved navigation maps such as `powerplant_local.pcd`. |
| `assets/validation/` | E2E high-speed validation/reference code and media. |
| `tools/` | Workspace, environment, model overlay, and control helper scripts. |
| `docker/` | Noetic build/runtime image helpers. |

## Build

```bash
cd /path/to/pipeline_inspection
export REPO_ROOT=$(pwd)
./tools/create_nav_integration_ws.sh
cd integration_ws
catkin_make -DCMAKE_BUILD_TYPE=Release
source devel/setup.bash
```

`integration_ws` is generated. Delete and recreate it after moving this project,
because catkin build files contain absolute paths.

## Main Interfaces

| Stage | Input | Output |
|---|---|---|
| Mapping | `/livox/lidar2` bridged to `/livox/lidar`, `/livox/imu` | `/Odometry`, `/cloud_registered`, Point-LIO PCD scans |
| Map tool | `third_party/point_lio/PCD/scans*.pcd` | `assets/maps/powerplant_local.pcd` |
| Localization | saved PCD, `/livox/lidar2`, optional `/livox/imu` | `/Odometry`, `/cloud_registered` |
| PX4 pose bridge | `/Odometry` | `/mavros/vision_pose/pose` |
| Global A* | saved PCD, `/Odometry`, `/move_base_simple/goal` | `/global_path` |
| Path target bridge | `/global_path`, `/Odometry` | `/e2e/local_target` |
| E2E PX4 controller | depth image, odom, target | `/mavros/setpoint_raw/attitude` |

## Mapping

```bash
source integration_ws/devel/setup.bash
source tools/use_env.sh
roslaunch navigation_bringup point_lio_mapping.launch gui:=false rviz:=false
```

After mapping is stopped cleanly, prepare the saved map:

```bash
rosrun map_tools prepare_pcd_map \
  --input_dir ${REPO_ROOT}/third_party/point_lio/PCD \
  --output ${REPO_ROOT}/assets/maps/powerplant_local.pcd \
  --voxel_leaf 0.2 --sor
```

For an end-to-end Noetic/PX4 smoke run, use:

```bash
tools/run_point_lio_sitl_smoke.sh
```

The script can bootstrap PX4 v1.11.0, install an `iris_mid360` SDF/model overlay,
build the Mid360 Gazebo plugin, build the integration workspace, start Point-LIO
mapping, prepare the saved PCD map, then relaunch saved-map localization and
record smoke artifacts under `assets/validation/point_lio_smoke/`.

### Legacy FAST-LIO Fallback

The original FAST-LIO launch remains available for comparison:

```bash
roslaunch navigation_bringup mapping.launch rviz:=false
```

FAST-LIO writes raw scans under `third_party/fast_lio/PCD/`.

### Pipe Factory World

For a denser factory-style pipe inspection scene, use the self-contained Gazebo
world `sim/worlds/pipe_factory_local.world`. It contains two long pipe racks,
cross-over headers, risers, storage tanks, pump skids, catwalks, and red visual
inspection markers.

```bash
source integration_ws/devel/setup.bash
source tools/use_env.sh
roslaunch navigation_bringup pipe_factory_mapping.launch rviz:=false
```

After flying/mapping the pipe factory scene, save its navigation map:

```bash
rosrun map_tools prepare_pcd_map \
  --input_dir ${REPO_ROOT}/third_party/point_lio/PCD \
  --output ${REPO_ROOT}/assets/maps/pipe_factory_local.pcd \
  --voxel_leaf 0.2 --sor
```

If you only need a quick static PCD map from the SDF geometry, generate it
without running Gazebo:

```bash
python3 tools/generate_pipe_factory_pcd.py
```

## Saved-Map Navigation

Start PX4/Gazebo with the same world and initial pose, then:

```bash
source integration_ws/devel/setup.bash
source tools/use_env.sh
roslaunch navigation_bringup global_astar_navigation.launch target_speed:=1.0
```

For the pipe factory map, launch:

```bash
roslaunch navigation_bringup pipe_factory_navigation.launch target_speed:=1.0
```

Publish a goal to `/move_base_simple/goal` from RViz. The default chain is:

```text
pcd_localization -> camera_pose_node
global_astar_planner -> global_path_target_bridge
e2e_px4_controller -> MAVROS/PX4
```

## Depth Camera Overlay

The E2E policy expects `/e2e/depth/image_raw`. Generate a local PX4 model copy:

```bash
./tools/install_depth_camera_model.sh
source tools/use_env.sh
```

Use model `iris_mid360_e2e` when launching PX4/Gazebo. The script writes under
`sim/models/` and does not edit the external PX4 tree.

## Baselines

The default navigation launch does not start FUEL, EGO/B-spline, or SE3 control.
They are kept for comparison and debugging:

- EGO/B-spline: `navigation_bringup/launch/ego_px4.launch` plus `bspline_target_bridge`.
- FUEL: `third_party/fuel/fuel_planner`.
- SE3 controller: `baselines/se3_controller`.
- PX4/MAVROS examples: `baselines/px4_real_control_example`.

Do not run SE3 and `e2e_px4_controller` at the same time, because both can
publish control setpoints.
